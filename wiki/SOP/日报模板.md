# 日报模板

> 每次日报输出后填写，便于追踪

## 格式规范（已确认，2026-04-19修正）

- header.title 用对象格式 `{"tag": "plain_text", "content": "标题"}`
- 内容放根级 `elements[]`（不是 body.elements）
- 不需要 schema: "2.0"
- 标题用 **加粗**
- 表格：用 markdown 表格（tag: "markdown"），不用原生 table 元素
- 区块间用 "---" 或 "━━━" 分隔
- 数字压缩（157万而非1,571,756）
- 结论单独拎出，不堆在表格里
- 策略建议用 ①②③④⑤ 编号

## 飞书卡片表格格式（2026-04-19重大修正）

**⚠️ 飞书 IM API 不支持原生 `tag: "table"` 元素，报 400 错误**
**✅ 正确做法：用 `tag: "markdown"` 承载 markdown 表格**

```json
// ✅ 正确：markdown 表格
{"tag": "markdown", "content": "| 景区 | 搜索指数 | 涨跌 |\n|------|---------|------|\n| 建业电影小镇 | 4,925 | 🔺 +14.32% |"}

// ❌ 错误：原生 table 元素（报400，从未通通）
{"tag": "table", "columns": ["景区", "搜索指数"], "data": [["建业", "4925"]]}
```

## 数据准确性原则（2026-04-15确认）

- **未确认不发送**：数据不完整时不发报告，先补全
- **禁止经验主义**：每项工作先读取全量原始数据，再分析判断
- **验证优先**：采集后两次验证一致性，确认再发群
