# 日报模板

> 每次日报输出后填写，便于追踪

## 格式规范（已确认）

- schema: "2.0"
- header.template: 根据内容类型（orange=数据、purple=分析、red=预警）
- 内容放 body.elements，每个元素 tag: "markdown"
- 区块间用 "---" 分隔
- 标题用 **加粗**
- 表格排版：窄表优先（≤5列），数字压缩（157万而非1,571,756）
- 结论单独拎出，不堆在表格里
- 策略建议用 ①②③④⑤ 编号

## 飞书卡片表格格式（2026-04-15确认）

**⚠️ 关键规则：columns 必须用字符串数组，不是对象数组**

```json
// ✅ 正确
{"tag": "table", "columns": ["景区", "搜索指数", "日环比"], "data": [["清明上河园", "196,617", "+16.68%"]]}

// ❌ 错误（会导致飞书报400）
{"tag": "table", "columns": [{"title": "景区"}, {"title": "搜索指数"}]}
```

## 数据准确性原则（2026-04-15确认）

- **未确认不发送**：数据不完整时不发报告，先补全
- **禁止经验主义**：每项工作先读取全量原始数据，再分析判断
- **验证优先**：采集后两次验证一致性，确认再发群
