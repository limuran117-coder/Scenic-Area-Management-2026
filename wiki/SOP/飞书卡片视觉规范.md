## 通用格式标准（所有任务统一执行）

### JSON结构
```json
{
  "schema": "2.0",
  "header": {
    "title": {"tag": "plain_text", "content": "📋 标题 | YYYY-MM-DD"},
    "template": "orange"
  },
  "body": {
    "elements": [
      {"tag": "markdown", "content": "━━━ **章节标题** ━━━"},
      {"tag": "markdown", "content": "内容..."}
    ]
  }
}
```

### 排版规则

| 元素 | 格式 | 示例 |
|------|------|------|
| 章节标题 | `━━━ **一、标题** ━━━` | `━━━ **一、背景** ━━━` |
| 编号列表 | `1️⃣ 2️⃣ 3️⃣` 或 `① ② ③` | `1️⃣ 内容...` |
| 要点 | `• 关键内容（**加粗**）` | `• 搜索指数 **4,925** ↑` |
| 数字高亮 | `**数字**` | `增长 **+68.6%**` |
| 风险标注 | `⚠️` | `⚠️ 风险提示` |
| 正面标注 | `🔥` | `🔥 增长亮点` |
| 趋势 | `↑ ↓ ➖` | `搜索 ↑ +14.32%` |
| 分隔线 | `---\n*数据来源：...*` | 放在卡片最底部 |

### 标题强化技巧
- 章节编号：`**一、**` `**二、**` `**三、**` 加粗
- 关键行：`🔥 **建业电影小镇**：双指标增速第一`
- 不要用纯文本标题，全部加粗或emoji引导

## 错误示范

❌ 单条长markdown塞所有内容
```json
{"tag": "markdown", "content": "标题\n---\n背景\n策略\n..."}  // 太长，不分段
```

❌ 表格拆成多个element
```json
{"tag": "markdown", "content": "| 表头..."},  // 表头和表格分开
{"tag": "markdown", "content": "| 数据行..."}   // 错误：应该放一起
```

❌ 没有章节分隔
```json
{"tag": "markdown", "content": "背景\n策略\n执行\n效果"}  // 没有分隔符，视觉混乱
```

## 正确示范

✅ 案例拆解格式
```json
{"tag": "markdown", "content": "━━━ **一、背景** ━━━\n\n• 内容..."}
{"tag": "markdown", "content": "━━━ **二、策略** ━━━\n\n**① 策略一**\n• 要点..."}
```

✅ 日报表格格式
```json
{"tag": "markdown", "content": "━━━ **竞品数据一览** ━━━"},
{"tag": "markdown", "content": "| 景区 | 搜索指数 | 搜索日环比 | 综合指数 | 综合日环比 |\n|------|---------|---------|---------|---------|"}
```
