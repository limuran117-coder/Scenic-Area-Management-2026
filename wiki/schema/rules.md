# Wiki Schema - 知识库结构规则

> 本文件定义 LLMWiki 的知识库结构规范，LLM 必须遵循

---

## 目录结构

```
wiki/
├── index.md              # 内容索引（必须）
├── log.md                # 操作日志（必须）
├── schema/               # 配置文件
│   ├── rules.md          # 本文件 - 结构规则
│   └── page-types.md     # 页面类型定义
├── 景区运营/             # 景区运营相关
├── 竞品分析/             # 竞品数据和分析
├── 营销策略/             # 营销方法论
├── 数据日报/             # 数据报告归档
├── 技术配置/             # 技术和工具配置
└── raw/                  # 原始资料（只读）
```

---

## 页面类型

### 1. 档案页 (Archival)
- 用途：记录静态信息、历史数据
- 命名：`<主体>基础档案.md`
- 示例：`电影小镇基础档案.md`
- 特点：长期有效，定期更新

### 2. 分析页 (Analysis)
- 用途：深度分析、洞察、趋势
- 命名：`<主体><主题>分析.md`
- 示例：`清明上河园深度分析.md`
- 特点：带时间戳，追踪变化

### 3. 方法页 (Method)
- 用途：SOP、操作流程、最佳实践
- 命名：`<主题>方法.md`
- 示例：`小红书运营方法.md`
- 特点：可执行、有步骤

### 4. 记录页 (Log)
- 用途：时间序列记录、操作日志
- 命名：`YYYY-MM-DD-<事件>.md`
- 示例：`2026-04-15-日报整合.md`

## Ingest 规则（新知识入库）

当有新的 raw 文件时，LLM 必须：

1. **读取 raw** - 原始资料放在 `raw/` 对应分类下
2. **写摘要** - 100字以内提炼核心信息
3. **更新实体页** - 创建或更新 `wiki/<分类>/` 下的页面
4. **维护索引** - 更新 `index.md` 中的对应条目
5. **记录日志** - 在 `log.md` 中追加操作记录

**级联规则**：一个来源可能更新 3-10 个页面

## 命名规范

- 使用中文标题
- 文件名用英文：`wiki/景区运营/cinema-town-basic.md`
- 链接用中文：`[[景区运营/电影小镇基础档案.md|电影小镇基础档案]]`
- 时间格式：`YYYY-MM-DD`

## Lint 检查规则

每月执行一次 lint 检查，查找：

| 问题类型 | 检查方法 | 处理方式 |
|---------|---------|---------|
| 矛盾点 | 全文搜索冲突关键词 | 标记并提出合并建议 |
| 过时信息 | 检查超过30天未更新的档案页 | 提醒更新 |
| 孤儿页面 | index.md 中未引用的页面 | 删除或补充索引 |
| 缺失概念 | 检查是否有遗漏的重要主题 | 提出新建建议 |

---

## Query 规则（查询）

- 优先读 wiki 页面，不是 raw 文件
- wiki 页面答案可存回 wiki 成为新页面
- 探索结果必须沉淀，不能只留在聊天记录

---

## 更新频率

| 页面类型 | 更新时机 |
|---------|---------|
| 档案页 | 数据有变化时 |
| 分析页 | 每周竞品分析后 |
| 方法页 | 有新 SOP 时 |
| 索引页 | 任何页面变更后 |
| 日志页 | 每次操作后追加 |
