# 画报 PS 模板设计规范书

> 交付：2026-05-14 | 版本：v1.0 | 给设计师

---

## 画布基础

| 参数 | 值 |
|------|-----|
| 尺寸 | **1080 × 1440 px** |
| 分辨率 | 72 dpi（屏幕显示） |
| 颜色模式 | RGB |
| 导出格式 | PNG-24（支持透明通道） |
| 文件名 | `模板A-有照片.png` / `模板B-无照片.png` |

> 1080×1440 是微信朋友圈图片最佳比例（3:4），不会被裁切也不会被缩放。

---

## 全局安全区（两个模板通用）

```
┌────────────────────────────── 1080px ──────────────────────────────┐
│  边距 40px                                               边距 40px │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │                                                              │  │
│  │                    核心内容区（1000×1280）                     │  │
│  │                                                              │  │
│  │                                                              │  │
│  │                                                              │⫶ 1400px
│  │                                                              │  │
│  │                                                              │  │
│  └──────────────────────────────────────────────────────────────┘  │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │              底部固定区：Logo + Slogan + 小程序码             │  │
│  └──────────────────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────────────────┘
```

**核心规则：**
- 所有动态文字（标题、数字、日期、文案）都落在**核心内容区**内
- 底部固定区 1400-1440px 预留给 Logo + 小程序码（PS 设计师需设计底部固定区，代码只覆写内容区文字）
- 边距 ≥ 40px，防止微信朋友圈边缘裁切

## 模板 A：有照片模式

### 使用场景
用户已上传照片（从相册选图或拍照），将照片嵌入画报。

### 布局结构

```
┌────────────────────────────────────────────────┐
│                                                │
│  标题区（y: 40-130）                             │  ← 动态：标题文字（白色/主题色）
│  ┌──────────────────────────────────────────┐   │
│  │                                          │   │
│  │                                          │   │
│  │         照片展示区（960×960）              │   │  ← 用户照片填充在此区域
│  │         左上角(60, 200)                   │   │
│  │                                          │   │
│  │                                          │   │
│  └──────────────────────────────────────────┘   │
│                                                │
│  元信息行（y: 1175-1220）                       │  ← 动态：分类emoji + 昵称 + 日期
│  ┌──────────────────────────────────────────┐   │
│  │  数字区（y: 1235-1370）                   │   │  ← 动态：大号数字 + DAYS
│  │                                          │   │
│  │  文案区（y: 1380-1420）                   │   │  ← 动态：一句话文案
│  └──────────────────────────────────────────┘   │
│                                                │
│  ═══════════ 底部固定区 ═══════════             │
│  Logo + Slogan + 小程序码（y: 1400-1440）      │
└────────────────────────────────────────────────┘
```

### 照片展示区详细尺寸

| 参数 | 值 |
|------|-----|
| 位置 | 左上角 (60px, 200px) |
| 尺寸 | **960 × 960 px**（正方形） |
| 圆角 | 24px |
| 边框 | 1px 白边，透明度 15% |
| 占位效果 | 照片未加载时显示浅灰底色 `#F0EDE8` |

### 设计师需要做的

照片展示区的**装饰层**可以包含：
- 照片四角的装饰角标（如半透明几何图形）
- 底部渐变遮罩（从透明到 `rgba(0,0,0,0.3)`），让白色标题可读
- 照片区外框装饰（细线+圆角+轻微投影）

**照片展示区不包含：** 标题文字、数字、日期、文案——这些都是动态数据，由代码叠加。

### 需要设计师导出的图层（分层 PNG）

| 图层 | 文件名 | 说明 |
|------|--------|------|
| 底层背景 | `bg-photo.png` | 满幅铺底（纯色或极简渐变） |
| 照片框装饰 | `deco-photo-frame.png` | 照片区域四角装饰+外框 |
| 照片区渐变遮罩 | `deco-photo-gradient.png` | 底部渐变（用于标题可读性） |
| 底部固定区 | `deco-bottom-bar.png` | Logo + Slogan + 小程序码占位 |

> 如果设计师觉得分层导出麻烦，也可以出**一张完整 PNG**（包含所有装饰元素），照片区和文字区留透明。

## 模板 B：无照片模式（纯设计）

### 使用场景
用户没有上传照片，画报靠设计感支撑。

### 布局结构

```
┌────────────────────────────────────────────────┐
│                                                │
│   分类标签胶囊（y: 160-220）                     │  ← 动态：BIRTHDAY / LOVE / ...
│  ┌──────────────────────────────────────────┐   │
│  │                                          │   │
│  │           中央卡片区                      │   │
│  │         （890 × 1060）                   │   │
│  │         左上角(95, 260)                   │   │  ← 装饰：圆角卡片+柔和阴影+光斑
│  │                                          │   │
│  │      [标题文字区 y: 330-420]              │   │  ← 动态：纪念日标题
│  │      [日期文字区 y: 430-480]              │   │  ← 动态：YYYY.M.DD
│  │                                          │   │
│  │          〰 〰 〰 〰 〰                   │   │
│  │                                          │   │
│  │      [数字大字区 y: 640-1020]             │   │  ← 动态：巨大数字 + DAYS
│  │         居中对齐                          │   │
│  │         字体：Helvetica Neue / 粗体       │   │
│  │         字号：约 200px                    │   │
│  │                                          │   │
│  │      [文案区 y: 1050-1150]               │   │  ← 动态：一句话治愈文案
│  │                                          │   │
│  │      [心情标签 y: 1180-1240]              │   │  ← 动态：满怀期待 / 岁月静好
│  └──────────────────────────────────────────┘   │
│                                                │
│  ═══════════ 底部固定区 ═══════════             │
│  Logo + Slogan + 小程序码（y: 1400-1440）      │
└────────────────────────────────────────────────┘
```

### 中央卡片详细尺寸

| 参数 | 值 |
|------|-----|
| 位置 | 左上角 (95px, 260px) |
| 尺寸 | **890 × 1060 px** |
| 圆角 | 72px |
| 背景 | 半透明白色 `rgba(255,255,255,0.72)` |
| 边框 | 2px `rgba(255,255,255,0.86)` |
| 阴影 | 外阴影，y偏移18px，模糊46px，颜色 `rgba(91,74,60,0.07)` |

### 卡片内装饰元素（设计师自由发挥）

| 元素 | 位置 | 效果 |
|------|------|------|
| 光斑1（粉） | 卡片左上角 (34, 40) | 半径约 460px，柔和粉色光斑，透明度 18% |
| 光斑2（蓝） | 卡片右上角附近 | 半径约 500px，柔和蓝色光斑，透明度 20% |
| 光斑3（青） | 卡片左下角附近 | 半径约 460px，柔和青绿色光斑，透明度 16% |
| 光斑4（金） | 卡片右下角附近 | 半径约 420px，柔和暖金色光斑，透明度 13% |
| 中心光斑 | 卡片正中 | 半径约 560px，主题色光斑，透明度 8% |

> 光斑效果参考：iOS 天气 App 背景的柔和模糊圆形。在 PS 中可以用「大尺寸软画笔 → 模糊 → 降低不透明度」实现。

### 需要设计师导出的图层

| 图层 | 文件名 | 说明 |
|------|--------|------|
| 底层背景 | `bg-classic.png` | 满幅铺底（柔和暖白或浅灰） |
| 光斑层 | `deco-blobs.png` | 5 个柔和彩色光斑 |
| 卡片层 | `deco-card.png` | 圆角卡片 + 边框 + 阴影 |
| 底部固定区 | `deco-bottom-bar.png` | Logo + Slogan + 小程序码占位 |

## 底部固定区规范（两个模板通用）

位于画报最底部 **y: 1380-1440**，高度 **60px**。

```
┌──────────────────────────────────────────────────┐
│                                                  │
│  [Logo]  NO FORGET · 记住每一个重要时刻  [小程序码] │
│  左边距40px     居中对齐           右边距40px      │
└──────────────────────────────────────────────────┘
```

| 元素 | 位置 | 说明 |
|------|------|------|
| Logo | 左侧 (40px, 1390px) | 品牌标识，约 40×40px |
| Slogan | 水平居中 | "NO FORGET · 记住每一个重要时刻" 字号22px 浅灰色 |
| 小程序码 | 右侧 (1040px, 1385px) | 留白方形区域 50×50px，代码动态填充 |

> 设计师只需在底部区域画出 Logo + Slogan 文字，留出小程序码的空白方形即可。

## 小程序码预留区

在底部固定区右侧，预留一个 **50×50px** 的白色方形区域。这个位置代码会调用 `wx.getUnlimitedQRCode` 生成的小程序码并叠加。

---

## 与开发的协作流程

1. 设计师按本规范设计 + 导出 PNG
2. 开发将 PNG 放入 `/static/posters/` 目录
3. 开发修改 `detail.js`，将 `renderSimplePosterClassic()` 中的 Canvas 手绘代码替换为 `ctx.drawImage('/static/posters/模板B-无照片/bg-classic.png', 0, 0, 1080, 1440)`
4. 动态文字部分保留 Canvas 绘制（约 30 行代码）
5. 联调测试 → 上线
