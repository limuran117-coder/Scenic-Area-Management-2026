# No Forget — 重要日期管家
## 产品需求文档 v1.1（对应代码 v17）

---

## 一、产品概述

**产品名称：** No Forget（不错过）

**一句话描述：** 一款记录和管理重要日期（生日、纪念日、忌日等）的倒计时小程序，帮助用户记住每一个值得铭记的时刻。

**核心场景：** 用户记录亲人和自己的生日、恋爱/结婚纪念日、宠物生日、孩子成长节点等，并在每个重要日期到来前收到提醒。

---

## 二、目标用户

- **主要用户：** 25-40岁的都市成年人，有家庭、有重要社交关系需要维护
- **核心需求：** 记住重要日子、不遗忘、不错过
- **使用动机：** 情感连接（记住重要的人）、自我管理（不要忘记重要的事）

### 3.2 添加/编辑页（pages/add/add）

**功能：**
- 创建新的重要日期，或编辑已有记录
- 表单字段：
  - 标题（必填，最多20字）
  - 日期（必填，年.月.日选择器）
  - 分类（必填，单选）
  - 是否循环（年度重复开关，默认开启）
  - 上传封面图（选填，可调用相册或相机）
  - 开始日期（选填，用于计算存活/已过去时间，如出生日期）
  - 提醒设置（提前N天提醒）
  - 文案模板（有心意的句子，继承分类预设）

### 3.3 详情页（pages/detail/detail）

**功能：**
- 完整展示单个倒计时项的所有信息
- 顶部：自定义导航栏（返回键 + "NO FORGET" 标题 + 占位）
- 有封面时：
  - 封面大图 + 渐变遮罩
  - 图上叠加：天数（⏳ +/− 天）+ 分类标签 + 标题 + 日期
  - 未上传照片时：显示上传照片区域（虚线框 + 📷 图标 + "上传照片"）
  - 已上传照片时：直接展示照片（不显示上传区）
  - 正文：倒计时文案（"第 XXX 天" 或 "还有 XXX 天"）+ 心意句
- 无封面时：
  - 已上传照片：直接展示照片（不显示上传区）
  - 未上传照片：显示上传照片区域
  - 玻璃数字卡：大号倒计时 + 分类 + 标题 + 日期
  - 正文：倒计时文案 + 心意句
- 底部固定操作栏：提醒 / 分享 / 编辑

**倒计时逻辑：**
- `isRecurring=true`（年度循环）：计算到今年或明年同月日的天数
- `isRecurring=false`（一次性）：计算到目标日期的天数
- 已过：显示 "第 XXX 天"（从 startDate 算起）
- 未到：显示 "还有 XXX 天"
- 精确倒计时：DD:HH:MM:SS，每秒刷新

### 3.4 我的页（pages/mine/mine）

**功能：**
- 主题切换器（10个主题的视觉选择器）
- 关于/帮助入口

## 四、数据模型

### 倒计时项（countdownItems）

```javascript
{
  id: String,              // 唯一ID，wx.guid
  title: String,           // 标题
  targetDate: String,      // 目标日期 "YYYY-MM-DD"
  categoryId: String,      // 分类ID
  isRecurring: Boolean,    // 是否年度循环，默认true
  startDate: String,       // 开始日期（用于存活计算）"YYYY-MM-DD"，可选
  coverImage: String,      // 封面图片本地路径，可选
  userPhoto: String,       // 用户上传的纪念照片本地路径，可选
  remindDays: Number,      // 提前提醒天数，-1表示不提醒
  direction: String,       // 方向：future/past（默认从分类继承）
  createdAt: Number,       // 创建时间戳
  updatedAt: Number        // 更新时间戳
}
```

### 存储结构

- `wx.setStorageSync('countdownItems', [...])` — 所有倒计时项
- `wx.setStorageSync('currentTheme', 'apple')` — 当前主题ID

## 六、技术实现

### 框架与工具
- **小程序框架：** 微信小程序（原生框架）
- **样式：** WXSS（微信样式表，CSS 子集）
- **逻辑：** JavaScript（ES6+）
- **存储：** wx.setStorageSync / wx.getStorageSync（本地 key-value）
- **无云服务、无后端、纯本地**

### 核心算法（utils/countdown.js）
- `getDiff(targetDate, now)` — 计算两个日期差（天）
- `getExactDiff(targetDate, now)` — 精确年月日时分秒差
- `getMainCountdown(item, now)` — 计算主倒计时（下一个目标日期）
- `getElapsedText(item, now)` — 计算已过去时间（存活计时）
- `getNextMilestone(date, daysLeft)` — 里程碑计算（100天/1000天等）

### 页面结构
```
pages/
├── index/      # 首页（卡片列表）
├── add/         # 添加/编辑页
├── detail/      # 详情页
└── mine/        # 我的（主题切换）
```

### Canvas 绘图（分享卡片）
- 使用 <canvas type="2d"> 同层渲染
- 在 detail.wxml 中隐藏绘制（left:-9999px）
- 支持生成分享图片

## 七、关键交互规则

1. **年度循环型**（生日/恋爱/结婚等）：每年同月日重复，跨越年份自动进位
2. **倒计时显示优先：** 详情页大字始终显示下一个目标日期的倒计时
3. **已过去显示：** 一次性已过事件显示 "已过 X 天"；循环型显示 "第 X 天"
4. **照片上传：** 有封面图时未上传照片显示上传区，已上传则展示照片；无封面时同理
5. **提醒设置：** 可提前N天设置提醒，存储在项数据中（实际提醒依赖微信订阅消息）
6. **主题切换：** 全局生效，存储在 localStorage

---

## 八、当前已知限制与待优化项

1. **倒计时负数问题：** 若存储的 targetDate 年份已过且非循环型，会显示负数
2. **分享功能：** canvas 绘制分享图功能存在但实际分享链路未完全打通
3. **提醒功能：** permission scope.notifyMsg 申请但可能未完成微信审核
4. **多端同步：** 数据仅存本地，切换设备后丢失
5. **数据导出/导入：** 无备份恢复机制
