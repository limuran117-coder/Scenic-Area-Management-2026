# 反馈捕捉机制 P0（2026-04-15强化版）

## 核心原则
- **双重捕捉**：用户反馈（被动）+ 自检脚本（主动）互补
- **触发时机**：用户说话 or 任务完成
- **捕获内容**：原始预测/实际结果/误差原因/认知更新
- **沉淀频率**：每周复盘Agent提炼SOP

## 捕捉层1：用户反馈（被动）
| 类型 | 触发词 | 记录重点 |
|------|--------|----------|
| 纠错 | 不对/不是/停止/糗了 | 哪里错、为什么、正确是什么 |
| 确认 | 对/很好/可以/就这样 | 什么做对了、可复用模式 |
| 建议 | 应该/不如/试试 | 改进方向、具体行动项 |

### 记录格式
```markdown
## [feedback/user] YYYY-MM-DD HH:MM

**类型：** 纠错/确认/建议
**场景：** 什么任务/场景下
**原始：** 我的判断/行为
**实际：** 真实结果/用户反馈
**原因：** 为什么会这样（如果知道）
**更新：** 认知更新/行为改变

**规则标签：** #rule/xxx
```

---

## 捕捉层2：自检脚本（主动）

**触发：** 每次isolated任务完成后自动调用

**评估维度：**
| 维度 | 权重 | 说明 |
|------|------|------|
| 内容完整度 | 40% | 数据字段是否齐全 |
| 数据准确度 | 40% | 数据是否在合理范围 |
| 格式正确度 | 20% | 飞书卡片/表格是否合规 |

**调用方式：**
```python
from self_check import self_check
result = self_check("douyin", data, card_valid=True, table_format=True)
```

**评分规则：**
- 总分 ≥ 0.8 → Skill有效，自动加分
- 总分 < 0.6 → 问题记录，进入纠错流程
- 总分 < 0.4 → 严重问题，触发告警

---

## Skill自优化规则

**触发条件：** 同一Skill连续3次评分 ≥ 0.8

**操作：** 自动优化该Skill的Prompt/规则
```markdown
1. 读取当前Skill的SKILL.md
2. 分析高分任务的共同特征
3. 提炼成功模式
4. 更新SKILL.md的"最佳实践"章节
```

**评分文件：** `memory/topics/feedback/skill_scores.json`

---

## Periodic Nudge规则（新增）

**触发条件：** 检测到以下情况时主动提醒用户

| 场景 | 提醒内容 |
|------|----------|
| 重要上下文被忽略 | "这个信息好像没用到，要记住吗？" |
| 规则冲突 | "发现两条规则矛盾，以哪个为准？" |
| 长任务无反馈 | "任务跑了很久了，需要我汇报进度吗？" |

**实现：** 嵌入每日复盘Agent的prompt

---

## 质量标准

1. 每个feedback必须包含「场景+原始+实际+更新」四要素
2. 自检评分 < 0.6 时必须记录issues
3. 每月汇总到 `wiki/SOP/feedback-distilled.md`
4. 高频错误优先制作成检查清单
